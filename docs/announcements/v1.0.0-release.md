## 1. Why OpenClawd Exists

The agent ecosystem is fragmenting into two camps: opaque cloud dashboards
that hold your keys, and raw SDK mazes that force you to reinvent the
plumbing every time. OpenClawd sits on the third path — **a self-hostable
monorepo where every component is inspectable, every secret is scoped, and
every agent is a first-class filesystem citizen.**

We wanted an environment where:

1. Installing an agent feels like installing a shell (`curl | sh` and done).
2. Running *multiple* agents on the same machine is trivial and safe.
3. Payments, tools, and identity are wired through open standards
   (x402, MCP, ACP) instead of bespoke SaaS APIs.
4. The aesthetic actually matches the cypherpunk ethos — ANSI colors,
   scanlines, matrix rain, and a lobster mascot that earns its place.

v1.0 is the point at which all four of those goals became real.

## 2. The Profiles System — Many Agents, One Box

The headline feature of v1.0 is the **profiles system** (`profiles/`),
inspired by the Hermes approach to identity-scoped runtimes.

Every profile is a directory under `~/.openclawd/profiles/<name>/` that
contains:

- **`CLAW.md`** — a markdown "soul charter" describing the agent's
  personality, constraints, and mission. Think of it as a system prompt
  that lives on disk and is version-controllable.
- **`config.yaml`** — model, provider, tool allow-lists, terminal theme,
  and runtime defaults.
- **`.env`** — secrets scoped to that profile only. No global leakage.
- An alias wrapper in `~/.local/bin/` so you can just type the profile
  name to launch it.

The `clawd-profile` CLI ships with the commands you'd expect from any
grown-up tooling layer:

```bash
clawd-profile create trader
clawd-profile clone trader trader-paper
clawd-profile use trader-paper
clawd-profile export trader > trader.tar.gz
clawd-profile list
```

The net result: you can run a paper-trading agent, a memecoin sniper, a
research assistant, and a DevOps bot **on the same machine**, each with
its own keys, its own soul, and its own budget — and swap between them
with a single command.

## 4. The Cypherpunk Installer

The `install.sh` script was completely rewritten for v1.0. It now opens
with a lobster ASCII animation, transitions through a matrix-rain
curtain, and lands on a clean success banner — all in pure bash, all
using a tight 256-color palette (LOBSTER `203`, NEON `118`, CYAN `51`,
MAGENTA `201`, VIOLET `141`).

Behind the aesthetics, it does real work:

- Detects shell (`zsh` / `bash`) and updates the correct rc file.
- Bootstraps a `default` profile so the first run is zero-config.
- Installs the `clawd-profile` binary into `~/.openclawd/bin/`.
- Ensures `~/.local/bin` is on `PATH`.
- Registers shell completions.

If you want to see it without installing anything, open the interactive
demo page at **[`docs/demo.html`](../demo.html)** — a standalone HTML
file with scanlines, a live install replay, a six-card onboarding grid,
and copy-to-clipboard snippets for every command in the release.

## 5. The Rest of the Stack

Profiles and sandboxes get the headline, but v1.0 also stabilizes the
surrounding fleet:

- **🤖 OpenAI Trading Agent** — a reference Solana trading agent wired
  to OpenAI tool-calls and x402 micropayments. The canonical example
  of "agent that earns, pays, and acts on-chain."
- **🌐 Cloudflare Workers Fleet** (`clawdrouter/`, `workers/`) — edge
  routers for MCP traffic and x402 settlement.
- **🧠 ACP Registry** (`acp_registry/`) and **AGENTS catalog**
  (`AGENTS/`) — machine-readable manifests so your agent can discover
  other agents.
- **🔐 Clawd Vault** (`clawd-vault-master/`) — Python-based secrets and
  policy engine, now with an example `policy.yaml` for per-tool allow
  lists.
- **🧩 Chrome Extension** — page agent + MCP bridge that turns any tab
  into a tool surface.
- **📲 WatchApp** — SwiftUI Apple Watch companion with complication
  views of your wallet state.
- **📟 CLI** — `clawd-cli.sh`, `clawd-connect.sh`, and registration
  helpers so your agent can announce itself to the network.

Every one of these modules has been in daily use internally. v1.0 is
the point at which we're comfortable telling external builders: *yes,
depend on this surface — we will version it properly.*

## 6. Security Posture

A few deliberate choices worth calling out:

- **No global `.env`.** Every profile has its own secrets directory.
  Compromising one profile does not leak keys from another.
- **Vault policy is declarative.** See
  `clawd-vault-master/policy.example.yaml` for how to allow-list tools
  per agent.
- **x402 payments** mean agents can transact without you handing them a
  funded hot wallet — they pay per call, from a scoped budget.
- **Everything is inspectable.** No closed-source binaries. No
  telemetry-by-default. If it runs on your machine, you can read it.

Full threat model and responsible-disclosure policy live in
[`SECURITY.md`](../../SECURITY.md).

## 7. Known Limits

- Windows is best-effort via WSL2. Native support is not a v1 goal.
- E2B templates require a valid `E2B_API_KEY`.
- The matrix-rain animation can be skipped with `CLAWD_NO_ANIMATE=1`
  for CI environments.

## 8. How to Get Started

```bash
# The one-liner
curl -fsSL https://raw.githubusercontent.com/x402agent/openclawd/main/install.sh | sh

# Then spawn your first named agent
clawd-profile create research
clawd-profile use research
research   # runs the agent under that profile
```

From there, read `profiles/README.md` for the full Hermes-adapted guide,
or crack open `docs/demo.html` for a visual tour.
