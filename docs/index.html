Flagship capabilities

The pieces that make OpenClawd more than the sum of its directories.

🦞 OpenClawd Orchestrator

Ties together wallets, Honcho memory, E2B sandboxes, MCP tools, and monetized runtime services.

🛣️ ClawdRouter

Routes across cloud and local models, supports hosted and local AI lanes, and sits on the payment-aware edge of the stack.

🌐 Browser automation & pAGENT

Wallet-aware browser-native operator surface for tool use and task automation, shipped as a Chrome extension.

💳 Payments as a primitive

x402, MPP, AP2, and A2A flows are first-class; agents can charge, pay, and settle on Solana.

🔐 ClawdVault

Hermès Vault Protocol gives every agent a vault-protected wallet at birth. Multi-sig & emergency recovery built in.

🧪 Formally verified skills

QEDGen Lean 4 proofs compile to proof_hash and land on-chain via the Solana Attestation Service.

🔑 Sign in with OpenRouter

OAuth PKCE flow — no client registration, no backend, no secrets. Drops OPENROUTER_API_KEY into ~/.openclawd/.env at agent birth.

🧠 AutoResearch loops

Agentic research and memory primitives wired through the agent bus and Claw3D integrations.

🏠 Local AI lanes

Routed and tunneled inference for self-hosted models, with documented fallback and arbitration paths.

Quick start

Clone the repo, copy env, install the repo-managed git hooks, and boot the orchestrator.

~/openclawd — bash
# 1. clone & configure $ git clone https://github.com/x402agent/openclawd.git $ cd openclawd $ cp .env.example .env # 2. install repo hooks & verify the machine $ npm run hooks:install $ npm run doctor ✓ all checks passed. # 3. install repo entry points & build the catalog $ npm run install:all $ npm run build:catalog # 4. start the orchestrator $ npm run dev:orchestrator 🦞 OpenClawd orchestrator listening on http://127.0.0.1:3000

Or use the bootstrap installer:

install.sh
$ bash ./install.sh

⛓️ Solana Attestation Service

Formally verified skills and agents — on-chain, trustlessly. Agents request QEDGen Lean 4 proofs, the proof hash lands as an SAS attestation, and any party can verify without a trusted intermediary.

Security guardrails

OpenClawd is meant to be cloned and forked publicly, so the repo ships built-in guardrails.

🪝 Pre-commit / pre-push hooks

Block staged .env, .pem, .key, and live-secret patterns. Rerun worktree + release hygiene before push.

🔍 Brand & doctor checks

Catch first-party branding drift in docs and verify the supported root bootstrap path.

📋 Release check

Validates public-release hygiene, package metadata, and required docs before tagging.
